# README 重写与截图更新设计

日期：2026-03-15

## 1. 目标

将项目 README 更新为与当前产品状态一致的 GitHub 首页文档，并替换为最新系统截图。

本次更新需要解决的问题：

- README 文案已经落后于当前功能状态
- 现有截图仍偏向旧页面结构和旧交互
- 多用户、值班人员管理、审计日志、批量导入、移动端移动模式等关键能力没有被完整展示

## 2. 范围

### 2.1 包含

- 重写 `README.md`
- 生成一组新的系统截图
- 使用新的截图替换 README 中过时的展示内容
- 更新 README 中的功能说明、权限模型、导入导出说明、部署入口说明

### 2.2 不包含

- 不修改业务代码
- 不为截图专门修改界面样式
- 不重做 banner，除非现有 banner 与新 README 明显冲突
- 不处理历史截图清理

## 3. README 结构

README 结构采用产品首页式组织：

1. Banner + Badge + 一句话定位
2. 核心能力
3. 最新系统截图
4. 适用场景
5. 快速开始
6. 登录与权限模型
7. REST API
8. 值班人员批量导入与导出能力
9. 部署说明入口
10. 技术栈
11. 项目结构
12. License

## 4. 文案更新重点

README 文案必须反映当前真实功能：

- 登录方式已是多用户账号体系，不再是单一共享密码模式
- 存在管理员和普通用户两类角色
- 值班人员管理已独立为页面，支持单位、类别、备注、筛选和批量导入
- 账号管理独立为页面
- 日志已扩展为审计日志，包含操作用户、IP 和来源
- REST API 已支持 Bearer Token
- 导出支持 CSV / JSON / XLSX
- 月历支持移动端“移动模式”
- 支持 VPS + GitHub Actions 部署

## 5. 截图策略

截图以当前产品的“页面化结构”为主，而不是旧的弹窗中心结构。

保留 6 张主截图：

1. 登录页
2. 排班主界面（月历）
3. 值班人员管理页
4. 设置或账号管理页
5. 审计日志页
6. 导出页

截图要求：

- 使用真实数据
- 文字清晰可读
- 尽量避免空白页
- 不使用旧的历史编号命名

建议文件名：

- `login-page.png`
- `dashboard-calendar.png`
- `duty-users-page.png`
- `settings-page.png`
- `audit-logs-page.png`
- `export-page.png`

## 6. 截图生成方式

使用本地运行的应用和 Playwright 生成截图：

- 通过登录态进入目标页面
- 使用固定视口
- 保持稳定数据状态
- 将截图输出到 `docs/screenshots/`

## 7. 风险与取舍

选择“重写 README + 更新截图”，而不是只补截图，是因为当前 README 文案本身已经落后，不更新结构会继续误导使用者。

选择保留现有 banner，是为了控制本次修改范围，避免把文档更新扩展成视觉重设计。
