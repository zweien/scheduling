# Issue #29 设计：换班日期跟随月历显示模式切换并优化移动端展示

## 背景

当前月历中，发生换班的日期会固定渲染为两行文本摘要：

- `原：XXX`
- `现：XXX`

这段展示逻辑直接绕过了现有 `displayMode` 分支，导致两个问题：

1. 顶部“切换为头像 / 切换为姓名”按钮对换班日期失效。
2. 移动端下，两行姓名摘要会明显挤压内容，姓名容易显示不全。

## 本次范围

本次只修显示层问题：

1. 换班日期也必须跟随全局显示模式切换。
2. 优化移动端下换班日期的展示紧凑度。

明确不包含：

1. 不改拖拽和换班交互流程。
2. 不改日志、数据库或 `original_user_id` 语义。
3. 不重做 tooltip 交互。
4. 不扩展到列表页、打印页和导出页。

## 目标

1. 换班日期在头像模式和姓名模式下都能正确展示。
2. 顶部显示模式切换对所有日期一视同仁。
3. 移动端下换班日期不再因为两行姓名文本而明显截断。

## 非目标

1. 不改变“原始值班员 / 当前值班员”的业务语义。
2. 不设计新的全局显示模式。
3. 不增加新的页面状态或本地持久化。

## 方案对比

### 方案 A：换班日期也纳入 `displayMode` 分支

姓名模式下显示文本版“原 / 现”，头像模式下显示头像版“原 / 现”。

优点：

- 和当前页面的切换逻辑完全一致。
- 改动集中在 `CalendarCell`，职责清晰。
- 不需要新增全局状态。

缺点：

- 需要维护两套换班日期的小布局。

### 方案 B：换班日期始终固定文本，头像模式只调整 tooltip

优点：

- 实现快。

缺点：

- 不能真正解决“切换头像不生效”。
- 只是视觉补丁，不符合 issue 核心。

### 方案 C：移动端对换班日期强制使用独立简化模式

优点：

- 能缓解小屏拥挤。

缺点：

- 全局切换语义被打破。
- 不同日期表现不一致，学习成本更高。

## 选型

采用方案 A。

原因：

- 这是唯一能真正满足“切换头像仍生效”要求的方案。
- 变更范围最小，只影响单元格渲染分支。
- 兼顾桌面和移动端，不引入额外状态管理。

## 行为设计

### 未换班日期

保持现有逻辑不变：

- 头像模式显示头像
- 姓名模式显示姓名块

### 换班日期

判定条件保持不变：

- `schedule.original_user` 存在
- 且 `schedule.original_user.id !== schedule.user.id`

#### 姓名模式

- 桌面端继续显示：
  - `原：XXX`
  - `现：XXX`
- 移动端改为更紧凑的文本排版：
  - 缩小字号
  - 减少内边距
  - 让文本更早截断，但保留“原/现”标签和姓名开头

#### 头像模式

- 不再显示固定姓名摘要
- 显示两个小头像块，分别对应：
  - 原始值班员
  - 当前值班员
- 每个头像块用简短标签区分：
  - `原`
  - `现`

## 组件设计

### CalendarCell

这是唯一主要改动点。

当前问题是：

- 换班日期一旦命中 `showOriginalAndCurrent`，就直接渲染固定文本块，完全跳过 `displayMode`

改造后：

1. 保留 `showOriginalAndCurrent` 判定
2. 在该分支内部继续按 `displayMode` 细分：
   - `name` -> 文本版原/现
   - `avatar` -> 头像版原/现

### Tooltip

继续保留现有 tooltip。

原因：

- 窄屏下即使姓名模式压缩，tooltip 仍能提供完整信息
- 不需要额外新增详情弹层

## 样式设计

### 姓名模式

- 桌面端：
  - 维持两行布局
  - 背景层级清晰区分原/现
- 移动端：
  - 字号比当前更小
  - 行高更紧凑
  - 使用 `truncate`
  - 标签和姓名放在同一行，减少垂直占用

### 头像模式

- 两个头像垂直堆叠或上下紧凑排列
- 每个头像旁带一个极短标签：
  - `原`
  - `现`
- 避免显示完整姓名文本，降低拥挤风险

## 测试策略

### Playwright 回归

新增或扩展现有换班测试，覆盖：

1. 换班后切换到头像模式
   - 换班单元格不再包含 `原：` / `现：` 文本摘要
   - 同时仍有可识别的头像结构

2. 移动端下换班日期在姓名模式仍可读
   - 换班单元格仍显示 `原：` / `现：`
   - 不要求像素级布局断言
   - 只验证关键文本和模式切换结果

### 回归范围

至少保证以下旧行为不被破坏：

1. 普通日期头像/姓名切换仍正常
2. 换班日期“原始 / 当前”语义不变
3. 现有 tooltip 继续可见

## 风险与控制

### 风险 1：换班日期布局分支继续膨胀

控制：

- 把变化严格限制在 `CalendarCell` 内
- 不引入额外页面级状态

### 风险 2：移动端头像模式仍显拥挤

控制：

- 头像模式只保留短标签，不显示完整姓名

### 风险 3：测试过度绑定具体 DOM 结构

控制：

- 优先断言模式切换后的关键文本是否消失/出现
- 避免依赖像素尺寸或过细类名

## 实施顺序建议

1. 扩展换班显示的 Playwright 失败测试
2. 改造 `CalendarCell` 的换班显示分支
3. 跑显示模式回归与现有月历测试
