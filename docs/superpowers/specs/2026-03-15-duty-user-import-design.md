# 值班人员批量导入与模板下载设计

日期：2026-03-15

## 1. 目标

在值班人员管理页增加批量导入能力，提供 `.xlsx` 模板下载，并在导入前完成字段与内容校验。

本次需求的目标是：

- 提供标准模板，减少手工录入成本
- 在导入前明确拦截错误数据
- 避免重复人员记录
- 保持现有值班人员模型和页面结构不变

## 2. 范围

### 2.1 包含

- 值班人员管理页增加导入区域
- 下载 `.xlsx` 模板
- 上传 `.xlsx` 后解析和校验
- 校验通过后执行批量导入
- 按姓名判重并执行更新或新增
- 导入完成后刷新值班人员列表
- 记录导入汇总日志

### 2.2 不包含

- 不支持 `.csv`
- 不支持多工作表导入
- 不支持自定义列映射
- 不支持部分成功、部分失败
- 不支持历史模板兼容
- 不支持分页预览

## 3. 数据模型

导入目标仍然是现有值班人员表 `users`，不新增表结构。

导入涉及字段：

- `name`
- `organization`
- `category`
- `is_active`
- `notes`

判重规则：

- 使用 `name` 作为唯一识别键
- 若数据库中已存在同名人员，则更新资料
- 若不存在，则新增人员

## 4. 模板设计

模板只包含一个工作表，列固定，不支持映射。

字段定义：

1. `姓名（必填）`
2. `所属单位（必填，W/X/Z）`
3. `人员类别（必填，J/W）`
4. `是否参与值班（必填，是/否）`
5. `备注（选填）`

模板要求：

- 第一行为表头
- 第二行为示例数据
- 表头加粗和底色，直接标识必填和允许值
- 不额外增加说明 sheet，避免解析复杂化

## 5. 导入流程

### 5.1 下载模板

用户在值班人员管理页点击“下载模板”，服务端生成 `.xlsx` 文件并返回下载。

### 5.2 上传并预检

用户上传 `.xlsx` 文件后，前端将文件内容传给服务端进行解析和校验。

校验项：

- 文件是否存在工作表
- 表头是否与预期一致
- 必填字段是否为空
- `所属单位` 是否只包含 `W/X/Z`
- `人员类别` 是否只包含 `J/W`
- `是否参与值班` 是否只包含 `是/否`
- 同一导入文件内是否出现重复姓名

校验结果返回：

- 总行数
- 可导入行数
- 错误数
- 错误明细

### 5.3 确认导入

只有在校验完全通过时，前端才允许用户执行“开始导入”。

导入规则：

- 同名存在：更新
- 同名不存在：新增

导入后返回：

- 新增数量
- 更新数量

## 6. 服务端设计

### 6.1 新增模板构建模块

建议新增独立模块，例如：

- `src/lib/imports/duty-users-template.ts`

职责：

- 生成模板工作簿
- 返回二进制 Buffer

### 6.2 新增导入解析模块

建议新增独立模块，例如：

- `src/lib/imports/duty-users-import.ts`

职责：

- 解析上传的 `.xlsx`
- 校验表头和行内容
- 生成结构化校验结果
- 执行导入

这样可以把 Excel 解析逻辑和页面 action 解耦，避免把复杂逻辑堆进 action。

### 6.3 Action 扩展

在 `src/app/actions/users.ts` 新增三个 action：

- 下载模板
- 校验导入文件
- 执行导入

权限要求：

- 全部仅管理员可用

## 7. 前端设计

在 `src/components/DutyUserManagement.tsx` 新增一个导入卡片，放在当前新增/编辑区域附近。

UI 组成：

- 下载模板按钮
- 文件选择按钮
- 校验按钮
- 校验结果摘要
- 错误列表
- 开始导入按钮

交互规则：

- 未上传文件时不能校验
- 校验失败时不能导入
- 校验成功后才能导入
- 导入成功后刷新值班人员列表

## 8. 日志设计

导入成功后写入一条汇总日志。

建议动作：

- `import_users`

日志内容：

- `target`: `duty_users`
- `new_value`: 包含新增数和更新数

不记录逐行导入日志，避免噪音过大。

## 9. 错误处理

错误分为三类：

1. 文件错误
   - 无工作表
   - 非法文件内容

2. 模板错误
   - 表头不匹配

3. 数据错误
   - 必填缺失
   - 枚举值非法
   - 重复姓名

处理原则：

- 任何一条错误都阻止导入
- 错误信息必须带行号和字段名
- 前端只展示结构化错误，不自行猜测修复方式

## 10. 测试策略

### 10.1 单元/模块验证

- 模板生成字段正确
- 解析模块能识别合法文件
- 解析模块能拦截非法字段

### 10.2 浏览器回归

- 值班人员页可下载模板
- 上传非法模板时提示错误
- 上传合法模板后可完成导入
- 同名人员导入后更新而不是重复新增

## 11. 设计取舍

选择固定模板而不做动态列映射，是为了控制复杂度并减少导入错误面。

选择“全量通过才允许导入”，是为了避免半成功状态导致的排查成本。

选择按姓名判重，是当前场景下最小可行且对业务最直观的方式。
