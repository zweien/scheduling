# 月历视图桌面端默认显示姓名设计

## 背景

当前排班月历视图在所有端上默认使用头像模式。桌面端单元格空间更充足，默认显示姓名比头像更利于快速扫读排班结果；移动端空间有限，继续保留头像模式更合适。

本次改动只调整默认展示策略，不改变现有切换按钮，也不引入跨刷新持久化。

## 目标

- 桌面端首次进入月历视图时默认显示姓名。
- 移动端首次进入月历视图时默认显示头像。
- 用户仍可通过现有按钮在姓名/头像模式间切换。
- 不记忆用户选择，不跨刷新持久化。

## 非目标

- 不新增设置项。
- 不监听窗口尺寸变化后自动切换展示模式。
- 不修改列表视图、打印视图或导出逻辑。

## 方案

### 状态初始化

`CalendarView` 继续持有 `displayMode` 状态，但初始值不再固定为 `avatar`。

实现规则：

- 初始渲染先使用 `avatar`，保证服务端与客户端首帧一致。
- 组件挂载后基于客户端断点判断是否为桌面端。
- 若当前是桌面端，且用户尚未手动切换，则把 `displayMode` 修正为 `name`。
- 若是移动端，则保持 `avatar`。

断点判断复用当前 UI 的 `sm` 语义，使用 `window.matchMedia('(min-width: 640px)')`。

### 用户手动切换

继续保留当前切换按钮行为：

- 当前为头像模式时显示“切换为姓名”
- 当前为姓名模式时显示“切换为头像”

一旦用户在当前页面会话内手动切换，后续不再由默认策略覆盖。

### 为什么不监听 resize

窗口尺寸变化时自动切换会覆盖用户刚刚做出的手动选择，交互不稳定，也超出当前需求。首版只处理首次进入页面时的默认值，符合 KISS/YAGNI。

## 影响范围

- `src/components/CalendarView.tsx`
- 代表性 Playwright 用例

## 测试

- 桌面视口下进入日历页，默认按钮文案应为“切换为头像”，证明当前处于姓名模式。
- 移动视口下进入日历页，默认按钮文案应为“切换为姓名”，证明当前处于头像模式。
- 原有手动切换能力保持可用。
