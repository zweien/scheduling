# Issue #24 第二期设计：无排班月份空状态引导

## 背景

当前系统在没有排班数据时，月历视图仍然展示完整空网格，列表视图则显示一段简单的虚线框文案。  
这两种表现都能表达“暂无数据”，但都不足以引导用户完成下一步操作：

- 月历空网格缺少明确行动建议
- 列表空状态文案过于弱，不利于首次使用者快速理解下一步

## 目标

本次只完成 `#24` 的第二期能力：

1. 为“无排班月份”提供统一的空状态引导
2. 在月历视图和列表视图中复用同一套空状态语义
3. 提供明确的下一步操作入口：去生成排班

## 非目标

本次明确不包含：

- 搜索高亮
- 新的生成排班流程
- 空状态内嵌完整导入表单
- 新增插画资源管理系统
- 批量操作进一步扩展

## 交互设计

### 1. 月历视图

当当前可见月份范围内没有排班数据时：

- 不再优先展示空网格作为主反馈
- 显示一个居中的空状态卡片
- 卡片包含：
  - 标题：当前月份还没有排班
  - 说明：可以先生成排班，也可以导入已有值班表
  - 主按钮：`去生成排班`
  - 辅助提示：`也可以通过导入排班快速录入已有值班表`

### 2. 列表视图

列表视图也使用相同语义，但布局可更紧凑：

- 保持统一标题和说明
- 继续提供 `去生成排班` 主按钮
- 与月历视图共享同一个空状态组件

## 行为规则

- 只有在“当前视图没有排班数据”时显示空状态
- 一旦存在排班数据，恢复正常月历/列表展示
- 点击 `去生成排班` 时：
  - 优先聚焦或打开现有生成排班入口
  - 不新增新的生成页面或第二套生成流程

## 实现设计

### 1. 组件拆分

新增一个小组件，例如：

- [src/components/EmptyScheduleState.tsx](/home/z/scheduling/src/components/EmptyScheduleState.tsx)

职责：

- 展示空状态标题、说明和按钮
- 通过回调将“去生成排班”动作交还上层

不直接耦合具体业务逻辑，保持单一职责。

### 2. 页面集成

在以下组件接入：

- [src/components/CalendarView.tsx](/home/z/scheduling/src/components/CalendarView.tsx)
- [src/components/ListView.tsx](/home/z/scheduling/src/components/ListView.tsx)

设计要求：

- 共享同一套空状态组件
- 仅根据当前视图数据量决定是否显示
- 不额外引入新的全局状态

### 3. 与现有入口的关系

本次不重做排班生成入口。  
空状态中的 `去生成排班` 只负责将用户带到现有能力入口，例如：

- 打开当前已有生成排班对话框
- 或触发已有生成组件的可见状态

如果当前结构无法直接调起，也可以通过页面内跳转或可控回调实现，但不新增新的业务路径。

## 测试策略

### Playwright

- 当前月份无排班时，月历视图显示新的空状态
- 切换到列表视图时，也显示统一空状态
- 点击 `去生成排班` 后，现有生成排班入口可见

### 回归

- 有排班数据时不显示空状态
- 不影响月历/列表视图切换
- 不影响已有导入排班入口

## 方案取舍

本次选择“统一空状态引导组件”，原因如下：

- 收益明确，直接解决“无数据但不知道下一步做什么”的问题
- 和第一期底部浮动栏技术上几乎不耦合，适合作为独立第二期
- 不把搜索高亮混进来，范围清晰

这符合：

- KISS：只增强无数据场景的引导
- DRY：月历和列表复用同一个空状态组件
- YAGNI：不在空状态里内嵌完整业务流程
